遊ぶ環境を、きれいに分ける。

Mason Launcherは、Minecraft: Java Editionのプロファイル、Java、MOD、ログを整理する非公式インスタンスランチャーです。 必要なファイルは公式メタデータから取得し、起動できない原因を追いやすくします。

最新版を入手 リポジトリを見る

インスタンスを混ぜない

mods / config / saves / logsをプロファイルごとに分離し、VanillaとMOD環境の混線を防ぎます。

ランチャーの状態を、Materialの面で整理する。

画面は大きな装飾よりも、情報の階層・状態・行動の区別を優先します。

01 公式メタデータ

Minecraft本体やlibrariesは同梱せず、公式URLから取得・検証します。

 02 インスタンス分離

プロファイルごとにゲームディレクトリを分け、共有してよいキャッシュだけを共通化します。

 03 診断できる起動

Java、mainClass、classpath、stdout/stderr、latest.logを段階ごとに見える化します。

主要機能

カードは単一トピックに絞り、行動につながる情報だけを残します。

プロファイル管理

Vanilla、Forge、Fabric、NeoForgeなどの環境をカードで管理します。

Javaランタイム選択

Minecraftの要求JavaとOS architectureを確認し、起動前に不一致を検出します。

MOD / Modpack連携

現在のMinecraft versionとloaderに合うプロジェクトを検索し、対象インスタンスへ導入します。

正規認証

Microsoft、Xbox、XSTS、Minecraft Servicesの段階を分けて扱います。

日本語に強いUI

日本語を標準に、英語・繁体字・簡体字・韓国語にも対応します。

起動ログ保存

インスタンスごとのlauncher log、stdout/stderr、latest.logを確認しやすくします。

起動フロー

失敗時に「どこで止まったか」を曖昧にしないため、状態を段階に分けます。

認証と所有権確認
Microsoft OAuth / Xbox Live / XSTS / Minecraft Servicesを分離します。
公式ファイルを取得
version JSON、client.jar、libraries、assetsをhashとsizeで検証します。
Javaと起動引数を解決
required Java、natives、classpath、mainClass、JVM/game argsを組み立てます。
プロセスとウィンドウを確認
spawn、stdout/stderr、exit code、latest.log、画面表示を分けて記録します。

画面プレビュー

開発中ブランチのスクリーンショットです。仕様や見た目は変更される場合があります。

プロファイル一覧

バージョン、loader、MOD数、起動状態をまとめて確認できます。

プロファイル編集

メモリ、Java、Minecraft version、loaderを設定します。

Modrinth検索

MODとModpackをインスタンス条件に合わせて探します。

開発中のプロジェクトです。

Fabric / NeoForge、Modrinth Modpackのインポート、認証済みオフライン起動など一部の機能はBetaまたはExperimentalとして扱われます。
