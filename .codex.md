# Mason Launcher 作業メモ

## プロジェクト

- アプリ名: Mason Launcher
- npm識別子: `mason-launcher`
- 目的: Minecraft: Java Edition向けの非公式インスタンスランチャー
- 構成: Electron 42 / Vite / TypeScript / Material Web
- UI: フレームワークなしのDOM操作

## 基本方針

- Vanilla起動経路を壊さない。
- 認証回避、任意ユーザー名、所有権未確認での起動を実装しない。
- Minecraft、Forge、libraries、assets、MODを同梱しない。
- Microsoft/Minecraftのtokenをログへ出さない。
- rendererからNode.js APIを直接使用しない。
- ファイル操作、認証、プロセス起動、OS依存処理はmain processに置く。
- rendererとの通信はpreloadの`contextBridge`を経由する。

## 主要モジュール

- `src/main.ts`: Electron起動、設定、IPC、起動ワークフロー
- `src/auth-service.ts`: Microsoft / Xbox / XSTS / Minecraft Services認証
- `src/minecraft-service.ts`: ダウンロード、Java、Forge、起動処理の統合
- `src/minecraft-downloader.ts`: 公式メタデータとゲームファイルの取得・検証
- `src/minecraft-launch-resolver.ts`: classpath、JVM/game引数、mainClassの解決
- `src/minecraft-process-runner.ts`: Javaプロセスとログ、終了状態の監視
- `src/forge-service.ts`: Forge installer、libraries、processorsの処理
- `src/java-runtime-service.ts`: Java/JRE検出、選択、管理
- `src/modrinth-service.ts`: Modrinth APIとMOD同期
- `src/offline-auth-cache.ts`: 所有権確認済みアカウントのオフライン認可
- `src/diagnostics.ts`: ログ収集と秘密情報のマスキング

## 永続データ

- 現行設定: `%APPDATA%\Mason Launcher\launcher-settings.json`
- 現行管理runtime: `%APPDATA%\Mason Launcher\runtime\`
- 新規インスタンス: `<gameDirectory>\mason-launcher\profiles\<profileId>\`
- 旧アプリ名のuserDataは初回起動時に現行ディレクトリへ移行する。
- 保存済み`instanceDir`はセーブデータ保護のため変更しない。
- `.simple-craft-*`形式の隠しマーカーは既存環境との互換性のため維持する。

## 注意点

- `undici`は`7.27.2`固定。変更時は`src/xmcl-compat.ts`も検証する。
- Forge installerをMinecraft client.jarとして扱わない。
- Forge processor用classpathとゲーム起動用classpathを混同しない。
- オフライン起動は正規認証・所有権確認済みキャッシュが有効な場合だけ許可する。
- `productName`変更はElectronの`userData`パスを変更するため、移行処理を削除しない。
- `Modrinth`は外部サービス名として使用する。アプリ自身の名称には使用しない。

## 検証

変更後は最低限、次を実行する。

```powershell
npm run typecheck
npm run lint
npm test
npm audit --omit=dev
git diff --check
```

配布物に影響する変更では追加で実行する。

```powershell
npm run package
```

期待されるWindows出力:

```text
out\Mason Launcher-win32-x64\Mason Launcher.exe
```

## 2026-06-13 作業メモ

- `@xmcl/core`の`minecraft-window-ready`はOSウィンドウ検出ではなく、stdoutの初期化ログ検出として扱う。
- Windowsでは起動Java PIDと子孫PIDに限定して、可視・非最小化・画面内のトップレベルウィンドウを確認する。
- 実ウィンドウ未確認のコード0終了は`window-unverified`として正常終了扱いしない。
- 起動ログへ初期化トリガー行、ウィンドウプローブ結果、stdout/stderr/latest.log末尾を保存する。秘密情報は必ずマスクする。
- Javaインストールはアーカイブをストリーム保存し、SHA-1/SHA-256とサイズを検証してから展開する。
- 前回中断された`.staging-*`/`.download-*`は次回インストール前に削除する。実環境では不完全な展開残骸が2件見つかった。
- Foojay/BellSoftの実配布メタデータはSHA-1を返す場合があるため、SHA-256だけに限定しない。
- Electron renderer sandboxはmain processからspawnしたJavaを直接制限しない。`windowsHide`はコンソールウィンドウだけを隠す。
